THINK RDO REPOS ARE CONFUSING?

Don't know which to use?

OK, it'S not that confusing

  • Stable
  • Testing
  • Trunk

Stable

  • http://mirror.centos.org/centos/7/cloud/x86_64/openstack-<branch>/
  • Signed packages: Yes!
  • Source: released project tags (*)
  • Creation: manual tagging of packages in testing repo
  • Use for: production

(*) Unless your project is too lazy to release tags :P

TESTING

  • http://buildlogs.centos.org/centos/7/cloud/x86_64/openstack-<branch>/
  • Signed packages: No
  • Source: released project tags (*)
  • Creation: automated after spec change review, manual tagging
  • Use for:
    • Updates (if can't wait for stable)
    • Testing packages before going to stable

(*) Unless your project is too lazy to release tags :P

TRUNK

  • http://trunk.rdoproject.org/centos7-<branch>/
    • Where branch is [master-head|master|newton|mitaka|liberty]
  • Signed packages: No
  • Source: current upstream GitHub repo HEAD (*)
  • Creation: automated
  • Use for:
    • Updates (if can't wait for testing/stable)
    • CI jobs

(*) Exception for certain projects in centos7-master, that are pinned to their released versions in

upper-constraints.txt

You can help, too!

Different branches:

  • rpm-master: for trunk
  • newton-rdo: for stable/testing/trunk

Reviews at review.rdoproject.org
